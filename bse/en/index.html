BrickStoryEngine

With BrickStoryEngine you can play and create text-based stories in your terminal.

Download BrickStoryEngine (windows) Download BrickStoryEngine (linux)

Installation

To use BrickStoryEngine, download the excecutable with the button above or with the direct link at www.timopollarini.nl/bse/release/BrickStoryEngine.exe. BrickStoryEngine is now also available for linux! it is build using WSL Debian gnu/Linux 12. Every feature is tested.

IMPORTANT: Things stopping the program from running

BrickStoryEngine currently does not function properly without a .story file, this is being worked on. For now, you can download and use testStory.story. It is possible to replace, delete or make new situations and opions to make it your own. Put this file in your Documents folder, BrickStoryEngine searches for .story files in that directory. More about this in the section Open

AntiVirus

The program may be detected as a Trojan Virus by Windows Defender or other AntiVirus's, this is incorrect. Mark the program as safe and continue

Windows

Windows automatically blocks excecutables downloaded from the internet, fixing this requires a Certificate from Microsoft. This program does not have a Certificate. To bypass this, select the downloaded exe, press alt + enter (properties), click "Unblock" and click "OK" or "Apply".

General Navigation

You navigate BrickStoryEngine with menu's that display a title at the top and options below that. You navigate through the options with the arrow keys and you select a option with enter.

Main Menu

  • Play
  • Open
  • Edit
  • Exit

Play

When you select 'play', you enter the Play mode of BrickStoryEngine. Your Current story-file will be played. If no story-file is selected, you will be prompted to select one. More about story-files in Open. In play mode, you see a description of the situation at the top, for example 'He attacks, what do you do?'. Under the title are the options, for example 'Attack Back' and 'Run Away'. You navigate through the options with the arrow keys and you select a option with enter.

Open

Your stories are saved in .story files, to play or edit a given story file, you go to the Open menu. When you open the 'open' menu, the program will search through a folder for .story files. You then select between the found .story files. It searches for .story files in your Documents folder.

Edit

The edit menu is the most advanced, it contains many submenus

  • Back
  • Play
  • Save
  • Add Situation
  • Delete Situation...
  • x: Situation Title

Back

You return to the main menu with the back option, Your changes will be remembered until you close the program. To save your changes for the next time, select Save.

Play

You use this to quickly test your changes without having to go to the main menu. If you have reached the end of the story, you will return to the Edit menu

Save

This will save your story in the opened file. Warning: This will overwrite the previous version.

Add Situation

You create a new Situation with this option, follow these displayed steps:

  1. Situation Title: : In this step, you give the Situation a Title, this describes the situation the player is in. For example: He attacks, what do you do?.
  2. amount of options (default 2): : Here you decide how many options your situation will initially have, This can be 1 if the player does not have a choice, or 2, or 3. You can enter nothing and press Enter to automatically select 2 options. You can always edit, delete or make more options.
  3. Option text: : Here you enter the text that will be displayed as the first option, this can for exmaple be Attack back.
  4. Choose where this option will lead: : Here you choose to what situation this option will lead if selected. This includes situation number 0, which instantly ends the game.
  5. These last two steps will be repeated for the remaining options.

Delete Situation

Here you delete a Situation of choice, Problems may occur if there is still an option that refers to the then-removed Situation

x: Situation Title

Among the other options in the edit menu, there are the Situations, with the situation number on the left and the situation title on the right (see image). If you want to edit a situation or an option of it, select the situation (of which the option is). The situation edit menu contains several options:

  1. Edit Title
  2. New option
  3. Delete Option...
  4. y <- Optie titel

Edit Title

Met deze optie bijwerk je de titel van de geselecteerde Situatie. Het vraagt om de nieuwe titel en laat de vorige titel zien, voor als je het vergeten was. Als je toch niet de titel wil veranderen kan je gewoon op Enter drukken en niets invullen, dan blijft de titel hetzelfde.

New option

With this option, you create a new option in the selected Situation, follow these displayed steps:

  1. New option text: : Here you enter the text of the option, this will be shown to the player, for example this could be Attack Back.
  2. Select situation to refer to: : Here you can choose which situation this option will take the player to. This includes situation number 0, which ends the game immediately.

Delete Option...

This removes an option of your choice from the currently selected situation, note that this may cause problems if that was the only option.

y <- Option Title

y stands for the situation where this option will go, this is a number. For each option in a situation there is this option with the title of the option. if you select one of these options you edit that option with these steps.

  1. First you will be asked what you want to replace the option text with, you can leave this empty and press Enter if you only want to edit the re situation.
  2. Then you choose which Situation the option will go to.

Technical Specifications

BrickStoryEngine is an exe and can be run by double clicking the exe in your file manager (Explorer on Windows), this will open a Terminal window with BrickStoryEngine and the window will automatically close once BrickStoryEngine closes (either by Exit in the main menu or by an error.). It is also possible to run the exe from cmd, this will keep the terminal open even after BrickStoryEngine closes.

Supported command line arguments (parameters)

When running BrickStoryEngine from cmd or the Run Menu (Windows + R) it is possible to add arguments. There is currently one possible argument supported, file.story. Add the path to a .story file after BrickStoryEngine.exe, this can be a local file (e.g. file.story or ..\Documents\file.story), or a file somewhere else (e.g. c:\users\user\OneDrive\mystory.story). For a local file, the .story file must be in the same folder as BrickStoryEngine.exe (e.g. Downloads). When running BSE with this argument you will automatically be put into Play mode with that story.

Tips and Tricks

To easily play stories you can set it to play a story by double clicking on it, follow these steps:

  1. Right-click on a .story file > Open with > Choose another app
  2. Scroll down and click Choose an app on your PC.
  3. Go to the location where BrickStoryEngine is located (usually Downloads)
  4. Select BrickStoryEngine.exe and click Open
  5. Then select Always or Just once at the bottom.

Linux

Bse is also available for linux. it looks for .story files in /home/[username]/documents, if that folder does not exist, it just looks in your home directory. Don't forget to download the testStory.story file and put it in your documents/home folder as Bse does not yet function properly without a .story files.
